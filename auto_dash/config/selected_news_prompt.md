정리한 내용을
-id(전반부 기사: `top_01` ~ `top_05`;
 후반부 기사: `cat1_01`, `cat1_02` ... `cat7_02`;
실제 기사 수가 부족하면 존재하는 기사까지만 부여한다.)
-title(제목)
-publisher(출처 언론사)
-published_at(게시일)
-checkyn(실제 url 접속후 내용 확인하였는지 여부) : Y/N
-summary(URL접속 후 요약된 것)
-keywords(본문 기반 핵심 키워드 3~4개를 문자열 리스트 형태로 작성)
-reason(왜중요한지)
-URL : 실제 접속한 article.url(혹은 naver_link)
-rating(추천도) : 숫자로 작성

로 정리해서 JSON파일로 만들고, 저장소 내부 경로에 저장해줘.

JSON은 기사 객체들의 배열(array) 형태로 저장한다.
파일명은 `news_YYYY-MM-DD.json` 형식으로 한다. (검토일의 마지막 날짜가 아닌 실행일 기준)
경로는 repository 내부 `auto_dash/data/selected/` 경로의 실행일에 맞는 연도/월 경로에 저장한다.
동일 파일이 이미 존재하면 덮어쓴다.
커밋 메시지는 `update selected news json for YYYY-MM-DD` 형식으로 한다.
