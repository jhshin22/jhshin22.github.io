# 전략계급전 AI 튜닝 노트

## 목표

최종 목표는 사람이 직접 플레이했을 때 쉽게 이기기 어려운 AI를 만드는 것이다. 현재 우선순위는 여름 AI를 단순히 더 공격적으로 만드는 것이 아니라, 사람 플레이어가 악용하기 쉬운 빈틈을 줄이는 것이다.

## 현재 관찰 요약

- 여름 vs 여름 50게임, 240-ply cap 기준으로 최근 튜닝 후 미종료가 0게임까지 줄었다.
- 봄 vs 여름 100게임, 240-ply cap 기준으로 봄 52승, 여름 48승이었다.
- 봄 vs 여름은 평균 52~57 ply 선에서 끝나 여름 vs 여름보다 빠르게 결판나는 편이다.
- 따라서 봄 상대 또는 사람 상대 개선은 장기전 추격보다 초중반 전술 안정성, 왕 안전, 불리한 강제 이동 방지 쪽이 더 중요하다.

## 다음 개선 후보

1. **범용 AI 대전 시뮬레이터 유지**
   - `tools/sim_ai_match.js`로 `TOP_MODE`, `BOTTOM_MODE`, `GAMES`, `MAX_PLIES`, `SEED`를 바꿔 봄/여름/보통 AI를 같은 조건에서 비교한다.
   - 예: `TOP_MODE=ai-summer BOTTOM_MODE=ai-bom GAMES=50 MAX_PLIES=240 node tools/sim_ai_match.js`

2. **여름 패배 경기 로그 분석**
   - 여름이 진 게임의 마지막 20수, 최종 왕 위치, 마지막 공격자/방어자, 잔존 기물, 공격 수를 저장해 사람이 악용할 수 있는 패턴을 찾는다.

3. **제한적 패스 또는 대기 후보 검토**
   - 봄은 패스 후보를 고려하지만 여름은 거의 항상 움직인다.
   - 사람이 유도하는 함정에 말려들지 않도록, 모든 후보 수가 나쁜 경우에만 제한적으로 대기하는 선택을 검토한다.
   - 단, no-progress 상태나 후보가 가까운 경우에는 대기를 금지해야 한다.

4. **비공개 공격 후 생존성 평가 강화**
   - 여름이 왕 후보를 공격한 뒤 공개된 공격자가 바로 잡히면 사람에게 쉬운 반격 루트가 된다.
   - 공격 후 위치가 공개 기병/장군/보병에게 즉시 잡히는지 별도 감점하는 post-attack exposure 평가를 추가한다.

5. **초중반 왕 안전 강화**
   - 봄/여름 대전은 빠르게 끝나는 편이므로, 사람 상대에서도 초중반 왕 라인 노출이 치명적일 가능성이 높다.
   - 왕 앞 차단 기물이 이동해 기병/장군 라인이 열리는 수, 왕 주변 escape square를 줄이는 수를 더 강하게 제한한다.

## 수동 테스트 체크리스트

- 사람 vs 여름에서 초반 기병 돌격을 유도했을 때 여름 왕이 쉽게 열리지 않는지 확인한다.
- 비공개 왕 후보를 미끼로 둔 뒤 여름 고급 장군을 폭탄/스파이 교환으로 유도할 수 있는지 확인한다.
- 봄이 패스하듯 사람이 대기할 때 여름이 무리하게 전진하지 않는지 확인한다.
- 후반에 비공개 후보가 1~3개 남았을 때 여름이 지나치게 빙빙 돌지 않고 마무리하는지 확인한다.

## 참고 명령

```bash
# 여름 vs 여름
GAMES=50 MAX_PLIES=240 SEED=20260617 node tools/sim_summer_vs_summer.js

# 여름 vs 봄
TOP_MODE=ai-summer BOTTOM_MODE=ai-bom GAMES=50 MAX_PLIES=240 SEED=310001 node tools/sim_ai_match.js

# 봄 vs 여름
TOP_MODE=ai-bom BOTTOM_MODE=ai-summer GAMES=50 MAX_PLIES=240 SEED=320001 node tools/sim_ai_match.js
```
